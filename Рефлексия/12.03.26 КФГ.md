
Я узнал про TDD (Разработка через тестирование) — это методология, при которой код пишется только под тесты, а не наоборот.
у него ест 3 закона : 
1. Новый код пишется только для того, чтобы починить уже существующий падающий тест.
2. Теста пишется ровно столько, сколько нужно для падения (или не компиляции).
3. Кода пишется ровно столько, сколько нужно, чтобы этот тест прошел. Цикл занимает около 30 секунд.
