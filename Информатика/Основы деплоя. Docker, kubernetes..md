Деплой - процесс развёртывания приложения на хостинге, для обеспечения внешнего доступа. 

### Docker 

Docker - инструмент для контейнеризации, который упаковывает приложение с зависимостями в легкий изолированный  контейнер. Он изолирует процессы на уровне ядра и работает на одном хосте. Его задача увеличивать совместимость, упростить деплой и воспроизводимость. 

### Kudernetes 

Инструмент для управления контейнерами обеспечивающий высокую эффективность  отказоустойчивость развёрнутых систем. 

### Образ 

Образ Docker — это готовый «слепок» (шаблон) со всем нужным для запуска программы: кодом, библиотеками, настройками и командами. Он только для чтения.

### Контейнер 
### Суть процесса (3 шага):

1. **Есть Dockerfile** (рецепт)
2. **Собираем Образ** (образ)
3. **Запускаем Контейнер** (работающая копия образа)
### Пошаговая инструкция

#### 1. Скачайте проект (пример)

В терминале выполните:
bash
git clone https://github.com/docker/welcome-to-docker
cd welcome-to-docker
#### 2. Соберите Образ из Dockerfile
В папке с проектом выполните:
bash
docker build -t welcome-to-docker .
Объяснение команды:
- `docker build` — команда сборки образа.
- `-t welcome-to-docker` — дает образу имя (тег).
- `.` — текущая папка, где Docker ищет файл `Dockerfile`
#### 3. Запустите Контейнер
Способ А: Через терминал (CLI)
bash
docker run -p 8080:8080 welcome-to-docker
_(Порт может отличаться, например `-p 8089:8089`)_
Способ Б: Через Docker Desktop (GUI)
- Перейдите во вкладку **Images**.
- Нажмите **Run** (стрелка) напротив образа `welcome-to-docker`.
- В настройках укажите **Port number** (например, `8089`).
### Простое объяснение (Аналогия)
1. **Dockerfile** = Список ингредиентов и инструкций для пирога.
2. **Образ (Image)** = Сам запеченный пирог (лежит на тарелке, его можно хранить).
3. **Контейнер (Container)** = Вы отрезали кусок и едите его. Это **работающий** процесс.

### Dockerfile

## Как работает Dockerfile?
Docker читает файл построчно и выполняет инструкции. Каждая инструкция создаёт **новый слой** в образе.
### Процесс сборки (docker build):
1. Берётся **базовый образ** (например, Ubuntu, Python, Node.js)
2. Выполняется первая инструкция — создаётся **слой 1**
3. Выполняется вторая инструкция — создаётся **слой 2** поверх слоя 1
4. И так далее...
text: 
Слой 5 (CMD)         ┌─────────────┐
Слой 4 (COPY)        │  Финальный  │
Слой 3 (RUN npm)     │   образ     │
Слой 2 (WORKDIR)     └─────────────┘
Слой 1 (FROM node)         ▲
Базовый слой              слои
Преимущество слоёв:  
Docker кэширует каждый слой. Если вы изменили только 5-й слой, слои 1–4 не будут пересобираться — сборка происходит очень быстро.

Без него Docker — просто инструмент для запуска чужих контейнеров.  
С ним вы становитесь **автором** своих образов и полностью контролируете окружение для вашего приложения.
