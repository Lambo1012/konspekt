
## 1. Зачем это нужно?

Сортировка нужна, чтобы данные (числа, строки) были расположены по порядку. Это основа для поиска, анализа данных и работы баз данных.

## 2. Три золотых правила (Критерии)

Чтобы сравнить алгоритмы, смотрят на три вещи:

1. **Скорость (О-большое):** Как быстро растет время работы при увеличении количества элементов.
2. **Память:** Требуется ли дополнительный массив (память) для сортировки.
3. **Стабильность:** Меняются ли местами одинаковые элементы? (Важно, если сортируем сложные списки).
## 3. Топ-6 алгоритмов, которые нужно знать

### Группа А: Простые, но медленные (O(n²))

_Подходят только для маленьких списков (до пары тысяч элементов)._

1. Пузырьковая (Bubble Sort)

- **Как работает:** Соседние элементы меняются местами, если стоят неправильно. Самые большие "всплывают" в конец.
- **Скорость:** Очень медленная.
- **Суть:** Проще всего понять, но в реальном коде почти не используется.

2. Вставками (Insertion Sort)

- **Как работает:** Берем карту из колоды и вставляем ее в нужное место среди уже отсортированных.
- **Фишка:** Если массив _почти_ отсортирован, работает молниеносно (быстрее всех!).
- **Использование:** Часто используется в реальных программах для досортировки маленьких кусочков.

2. Выбором (Selection Sort)

- **Как работает:** Ищем самый маленький элемент и ставим его в начало. Потом ищем второй самый маленький и ставим следом.
- **Недостаток:** Нестабильный (ломает порядок одинаковых чисел).
### Группа Б: Быстрые и умные (O(n log n))

_Справляются с миллионами элементов. Основа программирования._

4. Быстрая (Quick Sort) — Король сортировок

- **Как работает:** (Разделяй и властвуй). Выбираем "главный" элемент (pivot). Все, кто меньше, летят налево, кто больше — направо. Потом сортируем левую и правую части по тому же принципу.
- **Плюсы:** Очень быстрая, не жрет память (работает внутри массива).
- **Минусы:** Нестабильная. Если выбрать плохой "главный" элемент — может тормозить.
- **Вердикт:** Стандарт для большинства задач (например, так работает `sort()` в JavaScript/Python).

5. Слиянием (Merge Sort) — Трудяга

- **Как работает:** Режем массив пополам до тех пор, пока не останутся одиночные элементы. Потом склеиваем (сливаем) их обратно, но уже по порядку.
    
- **Плюсы:** Всегда стабильная и предсказуемая скорость. Гарантия.
    
- **Минусы:** Требует дополнительной памяти (как копирка, чтобы склеивать).
    
- **Вердикт:** Используется в базах данных и везде, где важна стабильность.
    
6. Пирамидальная (Heap Sort) — Хитрец

- **Как работает:** Представляет массив в виде "кучи" (пирамиды), где наверху всегда самый большой элемент. Забирает его, перестраивает кучу, забирает следующий.
- **Плюсы:** Всегда быстрая и не требует лишней памяти.
- **Минусы:** Сложная для понимания, нестабильная.

## 4. Шпаргалка для экзамена (Самое главное)

| Алгоритм          | Скорость (в среднем) | Нужна ли память? | Стабильный? |
| ----------------- | -------------------- | ---------------- | ----------- |
| **Пузырьковая**   | Очень медленно (n²)  | Нет              | Да          |
| **Вставками**     | Медленно (n²)        | Нет              | **Да**      |
| **Выбором**       | Медленно (n²)        | Нет              | Нет         |
| **Быстрая**       | **Быстро** (n log n) | Нет (экономит)   | Нет         |
| **Слиянием**      | **Быстро** (n log n) | Да (тратит)      | **Да**      |
| **Пирамидальная** | **Быстро** (n log n) | Нет              | Нет         |

- n² — значит время работы растет очень быстро (плохо).
- n log n — значит время работы растет медленно (хорошо).

## 5. Что отвечать на вопрос "Какую сортировку выбрать?"

1. "Если массив маленький" — Сортировку Вставками (просто и эффективно).
2. "Если массив большой и обычный" — Быструю (стандарт индустрии).
3. "Если важна стабильность (нельзя переставлять одинаковые)" — Слиянием.
4. "Если нельзя тратить дополнительную память" — Пирамидальную или Быструю.
